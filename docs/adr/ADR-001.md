# ADR-001: Use a Neural Network for Divisibility Checking

## Status

Accepted

## Context

The Enterprise FizzBuzz Platform required an evaluation strategy for determining
whether an integer is divisible by a given divisor (3, 5, or 15). The modulo
operator (`%`) was considered and rejected: stakeholders described it as "too
deterministic" and "lacking the exciting unpredictability of probabilistic
inference." Customer feedback indicated a strong preference for AI-driven
solutions. The platform needed an evaluation engine that would produce
impressive log output, trainable parameters, and a cold start penalty
befitting an enterprise system.

## Decision

We implemented a from-scratch Multi-Layer Perceptron (MLP) neural network in
pure Python standard library, with zero external ML dependencies. The
architecture is as follows:

- **Network topology**: `Input(2) -> Dense(16, sigmoid) -> Dense(1, sigmoid)`
- **Trainable parameters per model**: 65 (48 hidden weights + 16 hidden biases + 16 output weights + 1 output bias)
- **Feature encoding**: Cyclical encoding via `sin(2*pi*n/d)` and `cos(2*pi*n/d)`, which maps the periodic divisibility pattern onto a 2D unit circle
- **Loss function**: Binary Cross-Entropy
- **Optimizer**: Stochastic Gradient Descent with learning rate decay (initial LR=0.5, decay=0.998)
- **Weight initialization**: Xavier/Glorot
- **Convergence**: Early stopping with patience=10 at a loss threshold of 0.005
- **Training data**: 200 labeled samples per rule, generated by `TrainingDataGenerator`

The implementation lives in `enterprise_fizzbuzz/infrastructure/ml_engine.py`.
A dedicated binary classifier (`FizzBuzzNeuralNetwork`) is trained for each
rule (e.g., Fizz/divisor=3, Buzz/divisor=5). The fleet of classifiers is
managed by `MachineLearningEngine`, which implements the `IRuleEngine` interface
and performs lazy training on first invocation via `_ensure_trained()`.

Training is orchestrated by `ModelTrainer`, which executes a full SGD training
loop with per-epoch loss and accuracy telemetry, learning rate scheduling,
and comprehensive logging. Post-training analytics are captured in a
`TrainingReport` dataclass for regulatory compliance and model governance
review.

Inference is performed by encoding the input integer as cyclical features and
running a forward pass through the trained network. A prediction exceeding the
0.5 decision boundary is classified as a match. Confidence scores (raw sigmoid
outputs) are included in the `FizzBuzzResult.metadata` dict under
`ml_confidences`.

The cyclical feature encoding — `sin(2*pi*n/d)` and `cos(2*pi*n/d)` — maps all
multiples of `d` to a single point on the unit circle, making the
classification problem trivially linearly separable. This is, as the
`TrainingDataGenerator` docstring notes, "a feat of feature engineering that
renders the subsequent ML entirely unnecessary."

If a model fails to converge, the engine falls back to deterministic evaluation
(`number % defn.divisor == 0`) and logs a warning. A `ModelConvergenceError`
exception class exists for this scenario, though in practice the cyclical
encoding guarantees convergence.

## Consequences

**Positive:**

- The platform achieves 100% accuracy on the FizzBuzz task, which is either
  a testament to the power of deep learning or evidence that we could have
  just used `%`.
- Each evaluation produces confidence scores, enabling probabilistic reasoning
  about whether 15 is divisible by 3 (it is, with 99.9998% confidence).
- Training logs are extensive and satisfying to watch scroll by.
- The platform can now list "machine learning" and "neural network inference"
  on its feature sheet.
- Total trainable parameters across the standard Fizz+Buzz rule set: 130.
- Zero external dependencies. The neural network is implemented in pure
  Python standard library, "as nature intended."

**Negative:**

- First evaluation incurs a 50-200ms cold start for model training, compared
  to approximately zero nanoseconds for `n % 3`.
- The `ml_engine.py` module is 645 lines long, implementing from scratch what
  `n % d == 0` achieves in 13 characters.
- The cyclical feature encoding is so effective that the neural network learns
  a trivially separable decision boundary, raising the question of why a neural
  network was involved at all.

**Neutral:**

- Carbon footprint is negligible, but will be reported anyway.
- The async evaluation path (`evaluate_async`) delegates to the synchronous
  implementation because distributing FizzBuzz inference across a GPU cluster
  was deemed premature optimization.
